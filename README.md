## **🧭 Γενική ιδέα του προγράμματος (Φάση 1\)**

1️⃣ **Είσοδος (Input):**

* Το πρόγραμμα σε Java διαβάζει **ένα CSV** με τους βασικούς λογαριασμούς του προϋπολογισμού:

  * Κωδικό (π.χ. 0111, 0213\)

  * Περιγραφή (π.χ. «Φόρος Εισοδήματος», «Μισθοί Δημοσίων Υπαλλήλων»)

  * Κατηγορία (Έσοδα ή Έξοδα)

  * Ποσό (π.χ. 5.230.000 €)


➜ Εδώ η Java είναι “κύριος ελεγκτής”: διαβάζει το CSV, επαληθεύει τα δεδομένα (σωστά ποσά, κατηγορίες κ.λπ.) και τα αποθηκεύει προσωρινά (σε λίστες ή βάση δεδομένων αν θέλετε επιμονή).

---

2️⃣ **Επεξεργασία (Processing):**

* Η Java κάνει:

  * Ομαδοποίηση (π.χ. σύνολο εσόδων, σύνολο εξόδων)

  * Έλεγχο ισορροπίας (αν ο προϋπολογισμός είναι πλεονασματικός/ελλειμματικός)

  * Υπολογισμό δεικτών (π.χ. ποσοστό εσόδων από φόρους)

* Παράλληλα επιτρέπει αλλαγές (edit/delete) σε συγκεκριμένες γραμμές, π.χ. αν αλλάξει ένα κονδύλι.

---

3️⃣ **Ανάλυση & Οπτικοποίηση (Visualization):**

* Εδώ μπαίνει **η Python**:

  * Δημιουργεί **διαγράμματα πίτας** (κατανομή εσόδων, εξόδων)

  * **Ραβδογράμματα** για εξέλιξη ετών ή κατηγοριών

  * Ενδεχομένως **heatmaps ή trend lines** (αν υπάρχουν πολλαπλά έτη)

* Η Java απλώς **καλεί το Python script**, του δίνει π.χ. το CSV ή ένα ενδιάμεσο JSON, και παίρνει πίσω την εικόνα (`chart.png`).

---

4️⃣ **Αναφορές (Reports):**

* Η Java δημιουργεί **PDF αναφορές** (με `iText` ή `Apache PDFBox`):

  * Τίτλο: «Κρατικός Προϋπολογισμός 2025»

  * Πίνακα εσόδων/εξόδων

  * Βασικά γραφήματα που δημιούργησε η Python

  * Συνοπτική οικονομική ανάλυση (π.χ. "Τα έσοδα αυξήθηκαν κατά 3,2% σε σχέση με το 2024")

➜ Μπορεί να επιλέγει ο χρήστης ποια γραφήματα να συμπεριλάβει.

---

5️⃣ **Αποθήκευση & Επαναχρησιμοποίηση:**

* Μπορείτε να:

  * κρατάτε τα δεδομένα σε αρχεία CSV (αν είναι μικρή κλίμακα),

  * ή σε βάση SQLite (αν θέλετε αναζητήσεις και ιστορικό ετών).

* Έτσι κάθε χρόνο μπορεί να φορτώνεται ο νέος προϋπολογισμός και να συγκρίνεται με τον προηγούμενο.

### **🧩 Παράδειγμα ροής (conceptual)**

1️⃣ **Ο χρήστης ανοίγει το πρόγραμμα** και φορτώνει το αρχείο CSV του κρατικού προϋπολογισμού.

2️⃣ **Η Java**

* διαβάζει τα δεδομένα από το CSV,

* φτιάχνει τις κατάλληλες κλάσεις, αντικείμενα και μεθόδους

* υπολογίζει συνολικά έσοδα, έξοδα και ισοζύγιο.

3️⃣ **Ο χρηστης πλοηγείται στο κεντρικό μενού επιλογών**, όπου μπορεί να επιλέξει:

* **Επισκόπηση προϋπολογισμού** (γενική εικόνα εσόδων/εξόδων),

* **Αλλαγή κονδυλίου** (ποσοστιαία ή ποσοτική μεταβολή σε συγκεκριμένο κωδικό),

* **Προβολή γραφήματος**,

* **Εξαγωγή αναφοράς σε PDF**.

4️⃣ **Όταν πατηθεί “Προβολή γραφήματος”**, η Java εκτελεί εντολή:

python3 charts.py budget.csv

για να καλέσει την Python.

5️⃣ **Η Python**

* επεξεργάζεται τα δεδομένα,

* δημιουργεί διαγράμματα (π.χ. πίτα εσόδων, ράβδους εξόδων),

* αποθηκεύει τις εικόνες ως `esoda.png`, `exoda.png`, κ.λπ.

6️⃣ **Η Java**

* φορτώνει τις εικόνες και τις εμφανίζει στο UI,

* δίνει επιλογή **“Εξαγωγή σε PDF”**, όπου περιλαμβάνονται τα γραφήματα και οι αναλύσεις.

## **👤 3\. Θωμάς — Υπεύθυνος Μενού & Διεπαφής Χρήστη (JavaFX)**

**Ρόλος:** Σχεδιασμός και υλοποίηση του **κεντρικού μενού επιλογών** της εφαρμογής.

### **Πρέπει να κάνει:**

* Δημιουργεί το **κύριο μενού** (σε JavaFX ή Swing), με επιλογές όπως:

  * «Επισκόπηση Προϋπολογισμού»

  * «Αλλαγή Κονδυλίου»

  * «Προβολή Γραφημάτων»

  * «Οικονομική Ανάλυση»

  * «Εξαγωγή σε PDF»

  * «Έξοδος»

* Για κάθε επιλογή, το μενού πρέπει να:

  * Ρωτά τον χρήστη τι θέλει να κάνει.

  * Ανάλογα με την επιλογή, να καλεί τη σωστή μέθοδο Java:

    * π.χ. `BudgetOverview.show()`, `BudgetModify.updateItem()`, `ChartsCaller.runPython()`, `ReportGenerator.exportToPDF()`.

* Αν επιλεγεί **Αλλαγή Κονδυλίου**:

  * Ζητά **όνομα κατηγορίας ή κωδικό** (π.χ. "Έσοδα από φόρους").

  * Ρωτά αν η αλλαγή είναι **ποσοστιαία ή ποσοτική**.

  * Αν είναι ποσοστιαία → π.χ. “Πόσο % αύξηση/μείωση θέλετε;”

  * Αν είναι ποσοτική → “Πόσα € θέλετε να προσθέσετε/αφαιρέσετε;”

  * Ενημερώνει τη δομή δεδομένων και αποθηκεύει νέο CSV.

📌 *Εν ολίγοις:* Ο Θωμάς φτιάχνει το «κεντρικό πάνελ ελέγχου» του προγράμματος.  
 Πρέπει να φροντίσει η επικοινωνία με τον χρήστη να είναι καθαρή και κατανοητή (χωρίς τεχνικούς όρους).

## **👤 5\. Δημήτρης — Υπεύθυνος Εξαγωγής σε PDF**

**Ρόλος:** Δημιουργία τελικού PDF report του προϋπολογισμού.

### **Προτεινόμενη δομή:**

**Επιλογή χρήστη στο μενού:**

* «Εξαγωγή ενιαίου PDF» ή

* «Εξαγωγή σε 2 ξεχωριστά αρχεία»

  ### **Αν επιλεγεί Ενιαίο PDF:**

* Περιλαμβάνει:

  1. Πίνακα προϋπολογισμού (όπως τον βλέπει ο χρήστης).

  2. Διαγράμματα (εισαγόμενα από Python).

  3. Οικονομική ανάλυση (παραγόμενη από Άρη & Γ. Σαβαΐδη).

  4. Συνοπτικά σχόλια / συμπεράσματα.

  ### **Αν επιλεγούν 2 PDF:**

* **Αρχείο 1:** Προϋπολογισμός (πίνακας και αριθμητικά δεδομένα).

* **Αρχείο 2:** Διαγράμματα \+ Οικονομική ανάλυση \+ Σχόλια.

💡 Χρησιμοποίησε `iText` ή `Apache PDFBox` (Java) ή, εναλλακτικά, `reportlab` (Python) αν είναι πιο εύκολο.  
 Ο Δημήτρης θα πρέπει να συνεργάζεται στενά με:

* τον Γιώργο (Python) για τα paths των εικόνων

* και τον Άρη/Γ. Σαβαΐδη για το κείμενο της ανάλυσης.

  ## **👤 6\. Άρης & 👤 7\. Γιώργος Σαββαΐδης — Οικονομική Ανάλυση**

**Ρόλος:** Παράγουν **οικονομικά σχόλια** και **ερμηνείες** των αποτελεσμάτων του προϋπολογισμού.

### **Τι να εξετάζουν:**

1. **Ποσοστά σύνθεσης εσόδων:**

   * Π.χ. “Τα φορολογικά έσοδα αποτελούν το 65% των συνολικών εσόδων.”

   * “Τα δάνεια αντιπροσωπεύουν το 12% των συνολικών εσόδων.”

2. **Ποσοστά σύνθεσης δαπανών:**

   * Π.χ. “Οι δαπάνες μισθοδοσίας καλύπτουν το 40% των συνολικών εξόδων.”

   * “Οι επενδυτικές δαπάνες έχουν αυξηθεί κατά 10% σε σχέση με πέρυσι.”

3. **Ανάλυση ελλείμματος ή πλεονάσματος:**

   * Υπολογίζουν (Έσοδα \- Έξοδα).

   * Αν \< 0 → έλλειμμα → σύντομο σχόλιο για τη βιωσιμότητα.

   * Αν \> 0 → πλεόνασμα → σχόλιο για τη χρήση του.

4. **Αναλογία Δανείων / Εσόδων:**

   * Ερμηνεύουν το ποσοστό δανεισμού και τη σημασία του για το δημόσιο χρέος.

5. **Δείκτες τάσης (αν υπάρχουν δεδομένα πολλών ετών):**

   * Εξελίξεις εσόδων, δαπανών, ελλείμματος.

📊 *Παραδοτέο:* Κείμενο 3–4 παραγράφων με συγκεκριμένα στοιχεία και σχόλια, ώστε να ενσωματωθεί στο PDF από τον Δημήτρη.
